7.3 Set

Sơ lược về set

  • Không cho phép chứa phần tử trùng lặp.
  • Không duy trì thứ tự phần tử.
  • Các lớp triển khai phổ biến:
    • HashSet: Sử dụng bảng băm để lưu trữ, hiệu suất cao.
    • TreeSet: Lưu trữ phần tử theo thứ tự tăng dần, sử dụng cây đỏ-đen.

Các phương thức cơ bản

Thêm phần tử: add(E element) Set sẽ không thêm nếu phần tử đã tồn tại.
Code 

   import java.util.HashSet;
   HashSet<Integer> set  = new HashSet<>(); 
   set.add(10);  
   set.add(20);  
   set.add(10); // Bị bỏ qua vì trùng lặp
   System.out.println("Tập hợp: " + set);  //Tập hợp: [10, 20]
Xóa phần tử: remove(Object o).
Code 

   import java.util.HashSet;
   HashSet<Integer> set  = new HashSet<>(); 
   set.add(10);  
   set.add(20);
   set.remove(10);
   System.out.println("Tập hợp: " + set);  //Tập hợp: [20]
Tìm kiếm phần tử: contains(Object o) để kiểm tra sự tồn tại sẽ trả về giá trị boolean.
Code 

   import java.util.HashSet;
   HashSet<Integer> set  = new HashSet<>(); 
   set.add(10);  
   set.add(20);
   System.out.println(set.contains(10)); //true
Sắp xếp các phần tử:
  • TreeSet: Tự động sắp xếp theo thứ tự tự nhiên hoặc do Comparator cung cấp.
  • Khi tạo tree set thì nó sẽ tự sắp xếp, nên các bài toán liên quan tới Set cần sắp xếp thì chúng ta sẽ tạo TreeSet.
Code 

   import java.util.TreeSet;  
   TreeSet<Integer> set = new TreeSet<>(); 
   set.add(10);
   set.add(40);  
   set.add(30);
